# C Compiler — Compilers Project 2025-2026

A compiler that translates a subset of C into LLVM IR, built with Python 3 and ANTLR 4.

---

├── requirements.txt
└── src/
    ├── antlr_files/

    │   │   └── CParser/

## Requirements

- Python 3.10+
- Java (for ANTLR grammar generation)
- ANTLR 4.13.2 (`antlr-4.13.2-complete.jar`)

To visualise AST dot files:

```bash
sudo apt install graphviz xdot   # Ubuntu/Debian
brew install graphviz            # macOS
```

## Setup

### 1 — Install Python dependencies

```bash
pip install -r requirements.txt
```

### 2 — Generate ANTLR Files (one-time setup)

Run this from inside the grammar directory:

```bash
cd src/antlr_files/grammers/CParser
java -jar antlr-4.13.2-complete.jar -Dlanguage=Python3 -visitor CParser.g4
```

This generates `CParserLexer.py`, `CParserParser.py`, `CParserVisitor.py` and related files. Only needed again if you modify `CParser.g4`.

## Running the Compiler

All commands are run from the **project root** directory.

### Render the AST

```bash
python -m src.main --input src/antlr_files/example_source_files/test_types_vars.c --render_ast ast.dot
```

Then visualise with xdot:

```bash
xdot ast.dot
```

Or export to PNG:

```bash
dot -Tpng ast.dot -o ast.png
```

### Render the Symbol Table

```bash
python -m src.main --input src/antlr_files/example_source_files/test_types_vars.c --render_symb symb.dot
xdot symb.dot
```

### Compile to LLVM IR

```bash
python -m src.main --input src/antlr_files/example_source_files/test_printf.c --target_llvm output.ll
```

Run the generated LLVM IR:

```bash
lli output.ll
echo $?
```

Or compile to a native binary via clang:

```bash
clang output.ll -o my_program
./my_program
```

### Combine flags

You can combine multiple flags in one command:

```bash
python -m src.main --input input_file.c --render_ast ast.dot --render_symb symb.dot --target_llvm output.ll
```

### Disable constant folding

```bash
python -m src.main --input input_file.c --render_ast ast.dot --no-fold
```

## Supported C Features

### Types
- `int`, `float`, `char`
- Pointers (`int*`, `float**`, ...)
- `const` qualifier
- Arrays (1D and 2D): `int arr[5]`, `int matrix[2][3]`

### Operators
- Arithmetic: `+`, `-`, `*`, `/`, `%`
- Comparison: `==`, `!=`, `<`, `>`, `<=`, `>=`
- Logical: `&&`, `||`, `!`
- Bitwise: `&`, `|`, `^`, `~`
- Shift: `<<`, `>>`
- Unary: `+`, `-`, `!`, `~`
- Increment/Decrement: `++`, `--` (prefix and postfix)
- Address-of: `&`, Dereference: `*`
- Explicit cast: `(int)`, `(float)`, `(char)`

### Statements
- Variable declarations with optional initializer
- Array initializers: `{1, 2, 3}`, `{{1,2},{3,4}}`
- Assignments (including `*ptr =` and `arr[i] =`)
- `return`

### I/O (requires `#include <stdio.h>`)
- `printf` with format specifiers: `%d`, `%f`, `%c`, `%s`, `%x`
- `scanf` with `&var` arguments

### Optimisations
- Constant folding (compile-time evaluation of constant expressions)
- Constant propagation

### Semantic Analysis
- Undeclared variable detection
- Redeclaration detection (within same scope)
- `const` correctness (assignment to const variable)
- Type mismatch warnings (e.g. `float` to `int`)
- Pointer type compatibility checking
- Array initializer size checking
- `printf`/`scanf` format string argument count checking
- ISO C90 mixed declarations warning

## Reference Compiler

To compare output with GCC (C89 strict mode):

```bash
gcc -ansi -pedantic input_file.c
```

To generate LLVM IR with Clang:

```bash
clang -S -emit-llvm input_file.c
```
